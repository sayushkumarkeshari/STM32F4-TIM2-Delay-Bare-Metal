# STM32F446RE TIM2 Delay (Bare-Metal)

## Overview

This project demonstrates how to generate accurate software delays using the TIM2 hardware timer on the STM32F446RE microcontroller without using the HAL library.

The project is written in Embedded C using direct register-level programming (Bare-Metal) and was developed in Keil MDK.

The onboard LED connected to PA5 is toggled every one second using delays generated by TIM2.

## Working Principle

1. Enable GPIOA clock.
2. Configure PA5 as an output.
3. Enable TIM2 peripheral clock.
4. Configure the TIM2 prescaler.
5. Load the auto-reload register.
6. Start the timer.
7. Reset the timer counter before every delay.
8. Wait until the counter reaches the required delay value.
9. Toggle the onboard LED.
10. Repeat continuously.

---

## Timer Configuration

| Register | Value |
|----------|------:|
| PSC | 15999 |
| ARR | 0xFFFFFFFF |

With a 16 MHz timer clock:

- Timer Tick = 1 ms
- delay(1000) ≈ 1 second
